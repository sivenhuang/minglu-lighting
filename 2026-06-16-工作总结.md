## 三、做得不够好的

### ❌ 按钮样式反复调整
- Chat Now / TOP 按钮的位置、高度前后调整了 **7 次**才最终确定
- 原因：每次只改一个 CSS 属性，没有一次性全面评估
- **改进**：涉及多个联动元素的样式变更，应先阅读完整的相关 CSS，一次性规划方案，而非逐步试探

### ❌ 删除产品后未立即验证语法
- 多次用 sed 直接修改 JS 文件后未验证语法
- 虽然最终都通过验证，但存在引入语法错误的风险
- **改进**：每次修改 JSON/JS 数据文件后立即运行 `node -e "new Function(c)"` 验证

### ❌ 部分产品轮播图抓取不完整
- p106/p107/p115 的轮播图最初为空（Agent 分类规则把主图归到了 detail）
- 原因：部分源站页面的 alt 不以 "Xiuben" 开头，分类规则不够鲁棒
- **改进**：分类规则应加入降级策略——当 gallery 数量 < 3 时自动从 detail 补充前 N 张

### ❌ Cloudflare Pages 未完成自动部署
- 创建了 Pages 项目但未连接 Git，每次推送后仍需手动操作
- 当前实际仍由 GitHub Pages 处理（正确但不够自动化）
- **改进**：首次部署时应完成全流程，不留需要 "手动操作" 的尾巴

### ❌ 抓取流程可进一步优化
- 手动逐个 URL 调用 `playwright-cli` 效率偏低
- 每个产品需要 open → resize → scroll → eval → close 5 步
- **改进**：可封装为一个 Shell 函数或 Python 脚本，减少重复命令
